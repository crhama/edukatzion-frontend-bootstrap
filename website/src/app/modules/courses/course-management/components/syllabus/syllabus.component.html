Course Description

Databases are used to store corporate, government and private information. Securing this information involved following industry best practices and processes as well as writing secure code to interface with the database.

This course investigates the critical components of Oracle databases related to security, user privileges, roles and authentication. Database audit policies will also be reviewed, developed and tested.

Course Introduction

After completing this course, you should be able to:

  • compare and contrast database security architectures to secure and lockdown a database environment
  • use available authentication and authorization controls to implement and manage a secure database
  • design, develop, test, and document secure database applications to meet user requirements
